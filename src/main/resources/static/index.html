How the platform works

1. Login by role

Agents, supervisors, QA, clients, and admins sign in with seeded role-specific credentials.

2. Handle live work

Agents answer calls, create tickets, search the knowledge base, and share updates with the team.

3. Supervise operations

Supervisors review queues, internal chat, and report snapshots from the same backend data.

Real backend data

Tickets, knowledge base articles, queue state, and audit logs are served from Spring Boot and PostgreSQL-compatible storage.

Role-aware access

Each workspace is routed by role so agents, supervisors, QA, and admins see the correct tools.

Responsive interface

The layout adapts to desktop and mobile so the same system works in the office or on the move.
